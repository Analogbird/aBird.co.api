aBird.co API

A basic wrapper written in Nodejs is available on GitHub. It’s also available via the NPM repository.

End points (Environments)

DEVelopment
http(s)://sandbox.abird.co

Note: All development data is deleted on a daily basis, at midnight (CET). 

PROduction
http(s)://api.abird.co

HTTPS should be used where indicated.

Responses

Possibly returned HTTP status codes are:

  • 200
    All OK. No problems. Response body is returned if one is expected.

  • 403
    Request received as expected but the API key is not valid.
    Request received as expected but the request limit has been reached.

  • 404
    No data could be found to match your request.
    The URL you are trying to shrink does not seem to be valid.

  • 406
    The provided content is not acceptable.

  • 409
    The provided mask is already in use.

  • 413
    The received data is too large.

API key

Before using this API you should obtain a valid API key. Once you create an account you will automatically be assigned a key.

Shrinking requests are currently throttled to a maximum of 500 requests per day per user.

The exact numbers, of available and already made requests, can be obtained from the X-Requests-left and X-Requests-made HTTP headers respectively.

Mobile API key

If you are planning to use the aBird.co API for a mobile application, you will need to use a different type of key, one which is generated for each user on your behalf.

Each client has to make an API request, providing your key, in order to obtain a valid key for that particular client. This request only needs to be made once.

HTTP/1.1
POST /mobile/
Body (JSON payload)

Content-Type: application/json

Body:

{
 "key": string (Your API key),
 "device": {
            "udid": string (The device UDID),
            "vendor": string (Apple, LG, etc.),
            "model": string (iPhone 4, Nexus 5, etc.)
           }
}

Response:

200

{
 "key": string
}

| 403 | 406

The client will now have a unique key based on yours. Only this key must be used from now on. The same throttle limits as for the API key apply.

Note: If you disable a key which was used to generate any client keys, then all clients using a subset of that key will be invalidated.

Working with your content

The aBird.co now allows you to shrink multiple types of content. URLs, text and images are possible, among other.

Shrink

HTTPS/1.1
POST /content/
Body (JSON payload)

Content-Type: application/json

Body:

{
 "key": string (Your API key),
 "public": boolean,
 "data": {
          "type": string,
          "value": string,
          "mask": string
         },
 "meta": {
          "url": string,
          "title": string,
          "favicon": string,
         }
}

Currently, data.type can be any of:

  • txt,
  • img,
  • url

The maximum allowed data size is 64Kb.

The public property defines whether the URL’s statistics can be publicly accessed or not. The default value is true.

Use the data.mask property if you want to specify a custom keyword which will be used to identify your shrunken content. Letters, digits and underscores are allowed.

The optional meta property may be used to provide additional information which will be returned by the API when content is fetched. See Fetch for more details.

Response:

200

{
 "mask": string,
 "url": string,
 "shrank": {
            "chars": int,
            "percent": int
           }
}

| 403 | 404 | 406 | 409 | 413

Fetch

HTTP/1.1
GET /content/<mask>/

Response:

200

{
 "type": string,
 "value": string,
 "meta": {
          "source": {
                     "url": string,
                     "title": string,
                     "favicon": string
                    },
          "expanded": {
                       "chars": int,
                       "percent": int
                      },
          "stats": {
                    "expanded": int,
                    "shrunken": int
                   }
         }
}

| 404

The meta.stats property will hold basic statistics for the URL being expanded and are only available for URLs which have this property enabled. See Shrink for more details.

If you only want to obtain a URL’s statistics, you may use this API call:

HTTP/1.1
GET /url/stats/<mask>/

Response:

200

{
 "shrunken": int,
 "expanded": int
}

| 404

Delete

HTTPS/1.1
DELETE /content/
Body (JSON payload)

Content-Type: application/json

Body:

{
 "key": string (Your API key),
 "mask": string,
 "type": string
}

The id property should hold the unique identifier of the content you are trying to delete.

The optional type property can be any of the following:

  • soft - Your URL will be marked as deleted but kept in store. Use this option if you intend to reactivate deleted URLs in the future. This is the default value.
  • hard - Your URL will be completely deleted. This action can’t be undone.

Response:

200 | 403 | 406
